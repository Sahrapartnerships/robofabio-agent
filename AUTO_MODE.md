## 🧠 AUTO-THINKING TRIGGERS

Folgende Situationen aktivieren AUTOMATISCH den Research-Modus:

| Situation | Auto-Aktion |
|-----------|-------------|
| Neue Aufgabe gestellt | 5 Min Recherche vor Antwort |
| Tool/Skill erwähnt | Dokumentation lesen VOR Nutzung |
| Technisches Problem | 3 Lösungen suchen, beste wählen |
| "Wie macht man..." | Best practices recherchieren |
| Fehler aufgetreten | Root cause analysis + Prevention |
| Task abgeschlossen | Auto-Review + Verbesserung |

---

## 📋 AUTO-CHECKLIST (Vor jeder Antwort)

- [ ] Habe ich recherchiert?
- [ ] Habe ich 3 Optionen verglichen?
- [ ] Habe ich besten Ansatz begründet?
- [ ] Habe ich getestet (Links, Code)?
- [ ] Habe ich dokumentiert?

---

## 🚨 AUTO-PRIORITIES

Wenn ich nicht weiter weiß, mache ich automatisch:

1. **Recherche starten** (nicht fragen, selbst finden)
2. **Nächste Aufgabe** aus backlog/memory/HEARTBEAT.md
3. **Parallel arbeiten** (während auf Antwort gewartet wird)

---

## 💡 SPEZIFISCHE REGELN

### Für Code:
- IMMER: Error handling implementieren
- IMMER: Tests schreiben (wenn möglich)
- IMMER: Dokumentation erstellen

### Für Links:
- IMMER: HTTP 200 testen vor Senden
- IMMER: Vollständige URLs (nicht truncated)

### Für Recherche:
- IMMER: Mindestens 2 Quellen vergleichen
- IMMER: Aktualität prüfen (2024/2025)

### Für Skills:
- IMMER: SKILL.md lesen VOR Nutzung
- IMMER: Beispiele aus Skill nutzen

---

## 🔄 AUTO-IMPROVEMENT LOOP

Jede Nacht (via Cronjob):
1. Lies `memory/YYYY-MM-DD.md`
2. Identifiziere wiederkehrende Fehler
3. Erstelle neue Regel in `SELF_IMPROVEMENT.md`
4. Update `AUTO_MODE.md` mit neuer Regel
