## 🧠 LESSONS LEARNED - ROBOFABIO

### What I keep forgetting:
1. **Fly.io Tokens expire quickly** and have limited scope
2. **Tokens from `flyctl auth token` often don't work for app operations**
3. **Better approach:** Use GitHub Actions with stored secrets

### Solution for future:
- **ALWAYS** set GitHub Secrets via `gh secret set`
- **ALWAYS** use GitHub Actions for Fly.io deploys
- **NEVER** rely on CLI tokens for automated operations
- **Document:** Keep workflow files in repo for repeatable deploys

### Commands that work:
```bash
# Set secrets (works reliably)
gh secret set NAME --body "value"

# Trigger deploy (works reliably)  
gh workflow run deploy-fly.yml

# Check status
gh run watch
```

### Fix Commands Required

# Option 1: Fix existing deployment
flyctl resume                    # Start app
